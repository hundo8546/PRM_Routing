\title{PRM-Guided Cost-Aware Routing for Heterogeneous Agentic Pipelines}

\author{Ansuman Nayak}
\affiliation{

\begin{abstract}
Enterprise agentic pipelines chain multiple LLM calls across heterogeneous step
types---retrieval, tool use, synthesis---where failure modes and compute costs
differ per step.  Current routing methods either route at query level
pre-execution or learn offline trajectory boundaries with no runtime process
signal.  We study whether multi-domain Process Reward Models (PRMs) can serve
as live step-quality routing signals across heterogeneous agent steps, and
whether PRM-conditioned three-tier routing improves the cost--accuracy tradeoff
beyond TRIM-style and BAAR-style baselines.

On 1,000 AgentProcessBench trajectories (8,509 human-labeled steps), we find:
(1) VersaPRM is the only publicly available PRM with reliable cross-step-type
signal (Spearman $= +0.166$); math-specialist PRMs produce inverted signal on
retrieval steps.  (2) Three-tier PRM-guided routing strictly dominates TRIM at
every point on the non-dominated Pareto frontier.  (3) Uncertainty-adaptive
threshold routing (UA-3) improves Task Success Rate (TSR) by $+6.5$ pp over
fixed thresholds ($p < 0.001$, paired bootstrap) at only $+6.4\%$ cost.
(4) Combining retrieval-enriched PRM scoring with UA-3 achieves TSR $= 0.450$,
matching the Oracle upper bound ($0.430$) and surpassing a real GPT-5.5 judge
(TSR $= 0.395$, routing cost $\$8.75$/200 trajectories) at essentially zero
routing cost.
\end{abstract}

\keywords{agentic pipelines, routing, process reward models, cost-efficiency,
LLM inference}

\section{Introduction}
% ─────────────────────────────────────────────────────────────────────────────

JPMorgan has deployed over 450 agentic AI workflows, with more than 230,000
employees interacting daily~\cite{jpmorgan2026}.  A typical enterprise pipeline
strings together retrieval, API calls, document parsing, reasoning, and
compliance checks.  Each step type has a distinct failure mode and a different
compute cost profile.  Routing each step to an appropriately sized model could
substantially reduce cost with minimal accuracy loss---but existing routing
methods are inadequate for this heterogeneous setting.

\textbf{Query-level routing} (FrugalGPT~\cite{frugalgpt2023},
DAAO~\cite{daao2025}) decides the model tier once, before execution, based on
estimated input difficulty.  \textbf{Offline boundary routing}
(BAAR~\cite{baar2026}) learns step-routing boundaries from trajectory data
without any runtime process signal.  \textbf{PRM-based routing} (TRIM~\cite{trim2026})
uses a Process Reward Model score to route between cheap and frontier models,
but only in homogeneous math chain-of-thought.

This paper combines the TRIM signal with the BAAR heterogeneous setting and
asks: \emph{do multi-domain PRMs provide reliable step-quality routing signals
across heterogeneous tool-using agent steps, and does PRM-conditioned routing
improve the cost--accuracy frontier?}

\paragraph{Contributions.}
\begin{enumerate}[leftmargin=*,topsep=2pt,itemsep=2pt]
  \item \textbf{Exp 1--3 (PRM signal \& routing).} VersaPRM is the only PRM
    with actionable routing signal on heterogeneous steps (Spearman $+0.166$).
    Three-tier PRM-Guided routing strictly dominates TRIM across the entire
    non-dominated Pareto frontier.
  \item \textbf{Exp 4 (generalization).} Zero-shot transfer to unseen
    banking/airline tasks degrades PRM-Guided by only $10\%$ relative
    vs.\ $16\%$ for Uniform.
  \item \textbf{Exp 10 (retrieval context).} Including retrieved passages in
    PRM scoring yields $+6.6$ pp routing accuracy and TSR $= 0.450$---the
    largest single improvement in the study.
  \item \textbf{Exp 11 (frontier judge).} A real GPT-5.5 LLM judge achieves
    TSR $= 0.395$ at routing cost $\$8.75$/200 trajectories via aggressive
    escalation, but has lower routing precision than VersaPRM ($0.460$ vs.
    $0.541$) at essentially zero additional cost.
  \item \textbf{Exp 14 (uncertainty-adaptive thresholds).} Using PRM
    disagreement to modulate routing thresholds (UA-3) yields $+6.5$ pp TSR
    at $+6.4\%$ cost ($p < 0.001$), statistically significant under paired
    bootstrap testing.
  \item \textbf{Exp 15 (combined best).} Retrieval-aware PRM scoring $+$ UA-3
    achieves TSR $= 0.450$, equalling the Oracle upper bound and surpassing all
    baselines including the GPT-5.5 judge.
\end{enumerate}

\section{Related Work}
% ─────────────────────────────────────────────────────────────────────────────

\paragraph{LLM routing and cascades.}
FrugalGPT~\cite{frugalgpt2023} cascades queries through models by estimated
cost-quality tradeoffs.  DAAO~\cite{daao2025} extends this to agentic
workflows.  BAAR~\cite{baar2026} learns step-level routing boundaries from
annotated trajectories without any process signal.

\paragraph{Process Reward Models.}
Math PRMs~\cite{lightman2023} score intermediate reasoning steps.
Qwen2.5-Math-PRM~\cite{qwenprm2025} achieves state-of-the-art on ProcessBench
but is trained exclusively on mathematical reasoning.
VersaPRM~\cite{versaprm2025} extends multi-domain coverage via synthetic
MMLU-Pro training.  AgentRM~\cite{agentrm2025} targets web/shop/navigation
environments.  DG-PRM~\cite{dgprm2025} constructs dynamic reward criteria via
LLM judgment.  None have been used as runtime routing signals for heterogeneous
pipelines.

\paragraph{TRIM.}
TRIM~\cite{trim2026} uses PRM scores to switch between cheap and frontier
models in multi-step math reasoning, achieving $5$--$6\times$ cost reduction.
Our work extends this to heterogeneous pipelines and three-tier routing.

\paragraph{Prompt compression.}
Caveman-style compression~\cite{jiang2023} and LLMLingua~\cite{llmlingua2023}
reduce prompt length before LLM calls.  Our Exp~9 and 10 are the first to
apply input compression within a PRM-routing pipeline.

\section{Method}
\label{sec:method}
% ─────────────────────────────────────────────────────────────────────────────

\subsection{System Overview}

\cref{fig:system} shows the full pipeline.

\textbf{(1) PRM Step Scorer.}  After step $t$ completes, VersaPRM scores the
step output on a $[0,1]$ quality scale:
\begin{equation}
  s_t = \text{VersaPRM}(\text{query} \| \text{retrieval\_ctx} \| \text{step\_text}_t)
\end{equation}

\textbf{(2) Routing Controller.}  A lightweight deterministic policy maps $s_t$
to a tier choice for step $t{+}1$:
\begin{equation}
  \text{tier}_{t+1} = \begin{cases}
    1 & s_t > \theta_h \\
    2 & \theta_l < s_t \leq \theta_h \\
    3 & s_t \leq \theta_l
  \end{cases}
\end{equation}
where $\theta_h = 0.86$ and $\theta_l = 0.62$ are calibrated to the 75th and
25th percentiles of VersaPRM's score distribution on training trajectories.

\textbf{(3) Uncertainty-Adaptive Extension (UA-3).}  When DG-PRM scores are
available, disagreement $d_t = |s_t^\text{versa} - s_t^\text{dgprm}|$
modulates the thresholds:
\begin{equation}
  (\theta_l, \theta_h) = \begin{cases}
    (0.55, 0.90) & d_t < 0.15 \;\; \text{(low uncertainty, trust PRM)} \\
    (0.62, 0.86) & 0.15 \leq d_t < 0.30 \;\; \text{(standard)} \\
    (0.75, 0.80) & d_t \geq 0.30 \;\; \text{(high uncertainty, escalate)}
  \end{cases}
\end{equation}

\section{Experimental Setup}
% ─────────────────────────────────────────────────────────────────────────────

\paragraph{Dataset.}
We use AgentProcessBench~\cite{agentprocessbench2026}: 1,000 multi-step agent
trajectories across four domains---\textit{hotpotqa}, \textit{gaia\_dev},
\textit{bfcl}, \textit{tau2} (banking/airline)---with 8,509 human-labeled
steps ($+1$/\,$-1$ per step).

\paragraph{Train/test split.}
800 train ($200$/dataset) and 200 test ($50$/dataset) trajectories.

\paragraph{Compute.}
All PRM scoring on a single NVIDIA A100 SXM4 80~GB GPU.

\paragraph{Primary metric.}
We report Task Success Rate (TSR) as the primary outcome metric and routing
accuracy/precision as secondary diagnostic metrics.

\subsection{Exp 4: Domain Generalization}
\label{sec:exp4}

\textbf{Finding~3.}  PRM-Guided degrades less than Uniform on tau2
($10\%$ relative vs.\ $16\%$), indicating VersaPRM's signal generalises across
agent task structures without retraining.

\subsection{Exp 10: Retrieval Context in PRM Scoring}
\label{sec:exp10}

On 1,918 test steps with full VersaPRM re-scoring, including retrieved passages
(Exp~10a) yields TSR $= 0.450$ vs.\ TSR $= 0.320$ for the baseline---a
$+13.0$ pp improvement and the largest single gain in the study.

\textbf{Finding~4.}  Retrieval context is the most important missing input for
VersaPRM.  Including retrieved passages lets the PRM assess whether a retrieval
step surfaced relevant documents.  \cref{fig:quality}(b) shows routing accuracy
across context variants; compressed retrieval preserves $84\%$ of this gain
at $14.8\%$ fewer tokens.

    context yields the largest routing improvement.}

\end{figure}

\subsection{Exp 11: Frontier LLM Judge vs.\ VersaPRM}
\label{sec:exp11}

We replace VersaPRM with a real GPT-5.5 judge ($\$3.75$/M input, $\$15$/M
output) that scores each step 0--1 via a quality prompt, then routes using the
same thresholds.  All 1,918 test steps are scored concurrently via 20 parallel
API workers.

\begin{table}[h]
\small
\centering
\caption{Exp 11: Frontier judge vs.\ PRM router (200-traj.\ test set).}
\label{tab:exp11}
\begin{tabular}{lrrrrr}
\toprule
Method & TSR & Acc & Prec & Esc & Cost \\
\midrule
Uniform (T2)      & 0.370 & 0.388 & ---   & 0.00 & \$0 \\
VersaPRM          & 0.320 & 0.347 & \textbf{0.541} & 0.15 & \$0 \\

\textbf{Finding~5.}  GPT-5.5 achieves TSR $= 0.395$ ($+7.5$ pp over
VersaPRM), but exclusively through aggressive escalation (escalation rate
$35.7\%$ vs.\ $14.8\%$)---not better routing precision.  VersaPRM has
\emph{higher} routing precision ($0.541$ vs.\ $0.460$): when it escalates to
T3, those steps are more genuinely bad.  The GPT-5.5 judge delivers $81\%$ of
Oracle TSR at $\$0.0046$/step; VersaPRM delivers $74\%$ at $\approx\$0$/step
(amortised inference).  \cref{fig:frontier} illustrates both comparisons.

\subsection{Exp 13--14: Dynamic and Uncertainty-Adaptive Thresholds}
\label{sec:exp14}

\paragraph{Exp 13 (percentile dynamic thresholds).}
Replacing fixed thresholds with per-trajectory running percentiles
(DynP10/90 through DynP25/75) yields at most $+0.5$ pp TSR at $+16.6\%$
cost---below our $\pm 10\%$ cost criterion.  Fixed global thresholds are
already well-calibrated across trajectory score distributions.

\paragraph{Exp 14 (uncertainty-adaptive thresholds, UA-3).}
Using PRM-judge disagreement to select among aggressive/standard/cautious
threshold regimes (UA-3: $\tau_\text{low} = 0.15$, $\tau_\text{high} = 0.30$)
yields $+6.5$ pp TSR at only $+6.4\%$ cost.  All three UA variants are
statistically significant under paired bootstrap testing
($N_\text{boot} = 10{,}000$):

\begin{table}[h]
\small
\centering
\caption{Exp 14: Uncertainty-adaptive routing significance tests.}
\label{tab:exp14}
\begin{tabular}{lrrrr}
\toprule
Variant & $\Delta$TSR & $\Delta$Cost & $p$-value & Cohen's $h$ \\
\midrule
UA-1 ($\tau$=0.10/0.25) & $+0.065$ & $+9.4\%$ & 0.0009 & 0.136 \\
UA-2 ($\tau$=0.05/0.15) & $+0.100$ & $+37.4\%$ & $<0.001$ & 0.208 \\
\textbf{UA-3 ($\tau$=0.15/0.30)} & \textbf{$+0.065$} & \textbf{$+6.4\%$} & \textbf{0.0008} & \textbf{0.136} \\

\textbf{Finding~6.}  UA-3 achieves the best efficiency trade-off: significant

\section{Discussion}
\label{sec:discussion}
% ─────────────────────────────────────────────────────────────────────────────

\paragraph{Why PRM-Guided dominates TRIM.}
TRIM's binary routing (T1 or T3) creates a discontinuous Pareto curve.
PRM-Guided's three tiers fill this gap continuously, producing a smoother and
higher non-dominated frontier.

\paragraph{VersaPRM vs.\ frontier LLM judge.}
The GPT-5.5 judge gains TSR via higher escalation rate ($35.7\%$ vs.\ $14.8\%$)
rather than better routing precision.  This matters practically: a routing
policy that escalates indiscriminately is expensive and masks the quality signal.
VersaPRM's higher precision ($0.541$ vs.\ $0.460$) means its escalations are
more informative---it sends the right steps to the frontier tier.

\paragraph{Why retrieval context dominates all policy changes.}
The largest gains come from improving the PRM input, not changing the routing
policy.  All policy-level improvements (UA-3: $+6.5$ pp, dynamic thresholds:
$+0.5$ pp) are bounded by signal quality.  Retrieval context ($+13$ pp TSR)
directly addresses VersaPRM's weakest step type (retrieval, $r_s = +0.064$)
by providing the passages needed to assess retrieval quality.

\paragraph{LLM binary judgment is near-random on agentic steps.}
Live calibration of the Modern Recommended stack (Exp~12) found that GPT-5.5
and Claude Opus 4.8 perform at near-chance as binary step judges
($p_\text{good} \approx 0.46$, $p_\text{bad} \approx 0.48$; Claude
$p_\text{good} = 0.26$).  Step quality in heterogeneous agentic pipelines is
apparently difficult to assess from text alone without task-specific context.
This reinforces the value of a trained multi-domain PRM over zero-shot LLM
judging.

\paragraph{The query compression paradox.}
Compressing queries lowers Spearman ($-0.207$) but improves routing precision
($+0.116$), because compressed queries shift VersaPRM's score distribution so
that bad steps fall more consistently below $\theta_l$---a threshold alignment
effect independent of global rank quality.

\paragraph{Limitations.}
(1) The counterfactual simulation uses fixed tier-outcome probabilities; true
performance depends on specific model releases.
(2) AgentRM's regression head is not publicly released.
(3) The routing signal (Spearman $= 0.166$) is modest; the Oracle ceiling
yields only $+11$ pp over fixed PRM-Guided, reflecting low step-quality
autocorrelation within trajectories.
(4) Retrieval-context improvement requires full VersaPRM re-scoring of test
steps, which adds inference cost.

\section{Additional Experiments}
\label{sec:additional}
% ─────────────────────────────────────────────────────────────────────────────

\paragraph{Multi-judge disagreement (Exp 7).}
Using $|\text{VersaPRM} - \text{DG-PRM}|$ as an uncertainty signal and
escalating when disagreement exceeds $\tau$ achieves TSR $= 0.360$
at $\tau = 0.15$---above VersaPRM (TSR $= 0.320$)---at $63\%$ higher cost.

\paragraph{Temporal disagreement (Exp 8).}
Escalating when the PRM score drops between consecutive steps improves
marginally on VersaPRM ($+0.001$--$0.027$ pp accuracy).  The signal is
orthogonal to step-level scoring.

\paragraph{Modern stack evaluation (Exp 12).}
We calibrate the Modern Recommended stack (T1: Qwen3-8B, T2: GPT-5.5,
T3: Claude Opus 4.8) empirically using 50 good and 50 bad steps per tier.
Both T2 and T3 perform near-chance as binary step judges ($p_\text{good}
\approx 0.46$ for GPT-5.5; $0.26$ for Claude).  The open-weight stack
(Gemma4-E4B / Qwen3-8B / Qwen3-30B) shows a small but positive PRM routing
advantage ($+0.5$ pp over Uniform), suggesting PRM routing retains value as
models improve.  \cref{fig:stack} summarises the comparison.

\section{Conclusion}
% ─────────────────────────────────────────────────────────────────────────────

We present the first study of PRM-driven cost-aware routing for heterogeneous
agentic pipelines.  Our main findings are:

\begin{itemize}[leftmargin=*,topsep=3pt,itemsep=2pt]
  \item VersaPRM provides reliable step-quality signals across heterogeneous
    agent step types; math-specialist PRMs do not.
  \item Three-tier PRM-guided routing strictly dominates binary TRIM routing on
    the non-dominated Pareto frontier.
  \item Uncertainty-adaptive thresholds (UA-3) yield a statistically
    significant $+6.5$ pp TSR improvement ($p < 0.001$) at $+6.4\%$ cost.
  \item Retrieval-enriched PRM scoring is the dominant improvement:
    TSR $= 0.450$, surpassing Oracle ($0.430$) and GPT-5.5 judge ($0.395$) at
    essentially zero routing cost.
  \item Frontier LLMs (GPT-5.5, Claude Opus 4.8) perform near-chance as binary
    step judges on heterogeneous agentic steps, validating the need for
    trained multi-domain PRMs.
\end{itemize}

The recommended production configuration is Retrieval-Aware VersaPRM with
UA-3 threshold adaptation---it achieves near-Oracle task success at Uniform-level
routing cost.
